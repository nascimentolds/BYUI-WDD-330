Notes for week 7

Functions

The fact that functions are first-class objects means they can have properties and methods themselves.

Call and Apply methods

The call() method can be used to set the value of this inside a function to an object that is provided as the first argument.


The apply() method works in the same way, except the arguments of the function are provided as an array, even if there is only one argument.

Custom properties

There is nothing to stop you adding your own properties to functions in the same way that you can add properties to any object in JavaScript.

Memoization

A useful feature of this is that it provides result caching, or memoization. If a function takes some time to compute a return value, we can save the result in a cache property.

Immediately Invoked Function Expressions

An Immediately Invoked Function Expression– or IIFE is an anonymous function that, as the name suggests, is invoked as soon as it’s defined. This is easily achieved by placing parentheses at the end of the function definition An IIFE can be used to enclose a block of code inside its own private scope so it doesn’t interfere with any other part of the program.

Recursive Functions

A recursive function is one that invokes itself until a certain condition is met. It’s a useful tool to use when iterative processes are involved.

Event-driven Asynchronous Programming

Callbacks can be used to facilitate event-driven asynchronous programming.

Promises

A promise represents the future result of an asynchronous operation. Promises don't do anything that can't already be achieved using callbacks, but they help simplify the process, and avoid the convoluted code that can result from using multiple callbacks.

Async Functions

These functions are preceded by the async keyword and allow you to write asynchronous code as if it was synchronous. This is achieved by using the await operator before an asynchronous function.

Generators

These are special functions used to produce iterators that maintain the state of a value.

Currying

Currying is a process that involves the partial application of functions


AJAX

Ajax is a technique that allows web pages to communicate asynchronously with a server, and it dynamically updates web pages without reloading. Ajax allows JavaScript to request resources from a server on behalf of the client.

Cross-origin resource sharing (CORS)

It allows resources to be requested from another website outside the original domain.

Fetch API

Basic usage: Provides a global fetch() method that only has one mandatory argument, which is the URL of the resource you wish to fetch

Response interface

Deals with the object that’s returned when the promise is fulfilled. Response objects have a number of properties and methods that allow us to process the response effectively.

some properties:

  • headers
  • url
  • redirected
  • type

text()

The text() method takes a stream of text from the response, reads it to completion and then returns a promise that resolves to a USVSting object that can be treated as a string in JavaScript.

blob()

The blob() method is used to read a file of raw data, such as an image or a spreadsheet.

JSON response

JSON is probably the most common format for AJAX responses.

json()

The json() method is used to deal with these by transforming a stream of JSON data into a promise that resolves to a JavaScript object.
