Client-side form validation

To ensure all required form controls are filled out and in the correct format. Client-side validation should not be considered an exhaustive security measure. Validation done in the browser.

Form validation

When you enter data, the browser and/or the web server will check to see that the data is in the correct format and within the constraints set by the application

Main reasons for form validation

  • We want to get the right data, in the right format
  • We want to protect our users' data
  • We want to protect ourselves

Different types of client-side validation

  • Built-in form validation
  • JavaScript validation

Validation attributes

  • required
  • minlength / maxlenth
  • min / max
  • pattern

CSS pseudo-class

  • :valid
  • :invalid

required attribute

To make an input mandatory, add this attribute to the element. When this attribute is set, the element matches the :required UI pseudo-class and the form won't submit, displaying an error message on submission when the input is empty. Specifies whether a form field needs to be filled in before the form can be submitted.

minlength / maxlength attribute

Specifies the minimum and maximum length of textual data (strings)

min / max attribute

Specifies the minimum and maximum values of numerical input types

pattern

Specifies a regular expression that defines a pattern the entered data needs to follow.

Using Fetch

The Fetch API provides a JavaScript interface for accessing and manipulating parts of the HTTP pipeline, such as requests and responses. It also provides a global fetch() method that provides an easy, logical way to fetch resources asynchronously across the network.

A basic fetch request

fetch('http://example.com/movies.json')
.then(response => response.json())
.then(data => console.log(data));

Parameters

  • resource
  • init

Return value

A Promise that resolves to a Response object

Exceptions

  • AbortError
  • TypeError

Supplying request options

The fetch() method can optionally accept a second parameter, an init object that allows you to control a number of different settings
